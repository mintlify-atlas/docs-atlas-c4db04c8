grdpwasm is an open-source, plugin-free Remote Desktop Protocol client that runs the complete [grdp](https://github.com/nakagami/grdp) protocol stack compiled to WebAssembly inside your browser. This page explains what the project does, how its components fit together, and what you need before following the [Quickstart](/quickstart).

## What is grdpwasm

grdpwasm lets you connect to a Windows Remote Desktop server (or any RDP-compatible host) directly from a browser tab. There are no browser extensions, no native installers, and no server-side screen capture — the entire RDP client lives in a `.wasm` binary that the browser downloads and runs.

The project is built on two Go packages:

- **[grdp](https://github.com/nakagami/grdp)** — a pure-Go RDP implementation that handles the protocol, bitmap decoding, H.264, and audio.
- **grdpwasm** — the WASM wrapper that exposes `grdp` to JavaScript and pairs it with a lightweight proxy server.

## Architecture overview

Browsers cannot open raw TCP sockets, so grdpwasm uses a three-tier design:

```
Browser (WASM) ──WebSocket──► proxy (Go) ──TCP──► RDP Server
```

| Tier | What it does |
|---|---|
| **Browser (WASM)** | Runs the full RDP stack. Renders bitmaps to a `<canvas>`, decodes H.264 via WebCodecs, plays audio via Web Audio API, forwards keyboard and mouse events. |
| **Proxy (Go)** | Accepts WebSocket connections from the browser, dials the RDP server over TCP, and pipes bytes in both directions. Also serves the static WASM files. |
| **RDP server** | Any Windows host or RDP-compatible server reachable by the machine running the proxy. |

The proxy is necessary because the [WebSocket API](https://developer.mozilla.org/en-US/docs/Web/API/WebSockets_API) is the only way browsers can open a persistent, bidirectional byte stream to an arbitrary host. The proxy upgrades that WebSocket connection and forwards it as a plain TCP connection to the RDP port (default 3389).

## Key features

  <Card title="WebAssembly-powered RDP stack" icon="microchip">

  <Card title="Lightweight Go proxy" icon="arrows-left-right">

  </Card>
  <Card title="Remote audio via Web Audio API" icon="volume-high">

  </Card>
  <Card title="Full keyboard mapping" icon="keyboard">

  </Card>

  </Card>

## Requirements

Before you start, make sure you have the following:

| Requirement | Details |
|---|---|
| **Go 1.24 or later** | Required to build the WASM binary and proxy server. |
| **Reachable RDP server** | A Windows host or any RDP-compatible machine the proxy can reach over TCP. |
| **Modern browser** | WebAssembly support is required. WebCodecs is optional but recommended for H.264 hardware decoding (Chromium 94+, Edge 94+). |

<Note>
  The proxy accepts connections from any origin by default. Run it only on a trusted network, or place it behind a TLS-terminating reverse proxy (such as nginx or Caddy) before exposing it to the internet.
</Note>
