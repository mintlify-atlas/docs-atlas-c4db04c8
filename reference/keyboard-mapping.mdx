grdpwasm maps `KeyboardEvent.code` strings — the physical key identifiers produced by the browser — to RDP scan codes before sending them to the server. The mapping is defined in `keymap.go` and consulted by `jsCodeToRDP` on every `rdpKeyDown` and `rdpKeyUp` call.

## Scan code ranges

RDP uses two scan code namespaces:

- **Standard (0x00xx)** — the original PC/AT scan code set. Values in this range are sent as-is.
- **Extended / E0-prefixed (0xE0xx)** — keys that require an E0 escape prefix on the wire. These include navigation keys, the right-side modifier keys, and numpad variants. In `keymap.go` these are stored as `0xE0xx` integers so that the `grdp` library can distinguish them from standard codes.

<Note>
  Keys not present in the map return scan code `0` from `jsCodeToRDP` and are silently dropped — neither `KeyDown` nor `KeyUp` is sent to the server for unmapped codes.
</Note>

<Info>
  Five Japanese keyboard keys are included: `IntlRo`, `KanaMode`, `IntlYen`, `Convert`, and `NonConvert`. These are standard (0x00xx) scan codes from the Japanese 106/109-key layout and are ignored on keyboards that do not emit them.
</Info>

## Extended keys (E0xx prefix)

The following keys all carry the E0 escape prefix on the RDP wire. The `grdp` library reads the `0xE0xx` value from the keymap and encodes the prefix automatically:

`NumpadEnter`, `ControlRight`, `NumpadDivide`, `PrintScreen`, `AltRight`, `NumLock`, `Pause`, `Home`, `ArrowUp`, `PageUp`, `ArrowLeft`, `ArrowRight`, `End`, `ArrowDown`, `PageDown`, `Insert`, `Delete`, `ContextMenu`, `MetaLeft`, `MetaRight`
