grdpwasm is an open-source web RDP client that runs the [grdp](https://github.com/nakagami/grdp) protocol stack compiled to WebAssembly entirely inside your browser. A small Go proxy server bridges the browser's WebSocket connection to the RDP server's TCP port, giving you a full remote desktop in any modern browser tab.

## How it works

```
Browser (WASM) ──WebSocket──► proxy (Go) ──TCP──► RDP Server
```

Because browsers cannot open raw TCP sockets, grdpwasm ships a lightweight Go proxy that upgrades WebSocket connections from the browser and forwards them as plain TCP to your RDP host. The WASM module runs the complete RDP protocol stack — authentication, bitmap rendering, H.264 decoding, audio playback, and input forwarding — without any server-side screen capture.

## Get started

<Steps>
  <Step title="Clone and build">
    Clone the repository and run `make all` to produce the WASM binary, the Go runtime JS shim, and the proxy server binary.
  </Step>
  <Step title="Start the proxy">
    Run `./proxy/proxy -listen :8080 -static static` to start the WebSocket-to-TCP proxy and static file server.
  </Step>
  <Step title="Open the browser">
    Navigate to `http://localhost:8080`, fill in your RDP host details, and click **Connect**.
  </Step>
</Steps>

<Note>
  grdpwasm requires Go 1.24 or later and a reachable RDP server. See the [Quickstart](/quickstart) for full prerequisites.
</Note>
